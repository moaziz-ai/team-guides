The scripts you'll say most

Copy these verbatim. They're the 5 most common things you'll ever say.

#1
Start a Sooner project from your laptop
Say to Claude Code:
"Launch a new Draper project called <short-name> at ~/Sooner/<short-name>" Then open that folder in Claude Code Desktop to start building.
#2
Start a Sooner project from Telegram
Say to Draper on Telegram:
"Draper, set me up a Claude Code project for <what you want>." Draper proposes a name, you confirm, he runs the command. Open the folder in Claude Code Desktop.
#3
Check progress while away from laptop
Say to Draper on Telegram:
"How's <project-name> going?" Draper reads the live-status file and tells you exactly what happened.
#4
Ask Draper to build something in the background
Say to Draper on Telegram:
"Draper, just build <thing> in the background, I'm on the road. Ping me when done." Draper uses spawn mode (MODE 2). Returns with result.
#5
Declare a standing rule
Say anywhere (Telegram or Claude Code):
"From now on, on all Sooner projects, always use <rule>." Captured automatically. Dream cycle promotes to wiki if repeated 3+ times.

Starting a project at the laptop

Option A: Through Claude Code

Fastest path. You're already in Claude Code. Just say:

"Launch a new Draper project called broker-portal at ~/Sooner/broker-portal"

Claude Code runs this for you:

what runs openclaw-project launch --agent draper --pod sooner --project broker-portal --dir ~/Sooner/broker-portal

Then open that folder as a new project in Claude Code Desktop and start building.

Option B: Terminal directly

If you want to skip Claude Code and just run the command:

terminal cd ~/Sooner && openclaw-project launch --agent draper --pod sooner --project broker-portal --dir ~/Sooner/broker-portal && cd broker-portal && claude

For Axeed projects, swap the agent and pod

PodWhere it goesExample agents
Sooner~/Sooner/<name>draper, dinesh
Axeed~/Axeed/<name>elon, tariq, samir, zane, jad
Personal~/Personal/<name>iris (if applicable)

Starting a project from Telegram

The conversation flow

You →
Message Draper
"Draper, set me up a Claude Code project for <what you want>."
Alternatively: "Draper, plan and build <thing> for me, I'm on the road." (this routes to MODE 2 spawn)
Draper →
Proposes a name
"On it. I'll call it broker-portal at ~/Sooner/broker-portal. Sound good?"
You →
Confirm or tweak
"Yes go" · or · "Call it broker-sso instead"
Draper →
Runs the command, tells you how to open it
"Done. Open ~/Sooner/broker-portal in Claude Code Desktop or run cd ~/Sooner/broker-portal && claude in a terminal. My full context is loaded. Ping me for progress anytime."
Key insight: Whether you're on Telegram or in Claude Code, the folder you end up with is identical. Same CLAUDE.md, same hooks, same brief format. You're picking between two front doors to the same house.

Solo first, adopt later

Sometimes you start building alone in a random folder, then later decide the agent should own it.

You →
After building for a while, message the agent
"Draper, adopt ~/Sooner/pricing-work as pricing-page. I've been building solo, want you in the loop now."
Draper →
Runs the adopt command
"Adopted. All your existing files are preserved. I imported the last 3 Claude Code sessions you had on this folder. Hooks are now live - from here on everything gets logged. Anything you want me to focus on?"

Under the hood:

what runs openclaw-project adopt --agent draper --pod sooner --project pricing-page --dir ~/Sooner/pricing-work --note "Mo built solo, bringing Draper in"

Mid-build check-ins

While you're working in Claude Code, check progress from phone

Phone →
Message the owning agent on Telegram
"How's <project> going?" · or · "What have I done on <project> today?"
Agent reads live-status.md and answers with actual events (files changed, what you asked for, what got built).

Ask for a specific summary

"Summarize what was built on <project> this week."

While a background build is running, check status

terminal or ask Claude openclaw-project status --dir ~/Sooner/<project>

Picking up where you left off

Next day, open the project cold

You →
In a fresh Claude Code session on the project folder
"Read PROJECT_SUMMARY.md, tell me where we are, and propose the next step."
PROJECT_SUMMARY.md is auto-compiled after every substantive session. Always current. Claude picks up with full context in seconds.

Or ask the agent from Telegram first

"Draper, summarize where we are on <project> and what's left."
The magic of auto-compile: You don't have to track state across sessions. Every substantive Claude Code session auto-generates a fresh PROJECT_SUMMARY.md. Next session reads that first (30 seconds of context) instead of digging through the full log (hours of context).

Handoffs between modes

You were working in Claude Code, now want Draper to finish it

Close Claude Code (or just walk away). Message Draper:

"Draper, continue <project> in the background - <describe what's next>."

Draper runs openclaw-project continue which appends the new task to BRIEF.md and spawns Claude Code headlessly.

Draper was building in background, you want to take over

Message Draper:

"I want to take over <project>, I'll drive from Claude Code."

Or run yourself:

terminal openclaw-project takeover --dir ~/Sooner/<project> cd ~/Sooner/<project> && claude

The background Claude Code gets stopped cleanly. A TAKEOVER marker lands in the log. Your interactive session picks up with full visibility into what was done.

Stop a background build without taking over

"Draper, stop the background build on <project>."
or terminal openclaw-project stop --dir ~/Sooner/<project>

When Claude Code hits a limit mid-build

Claude Code has context window limits and per-session rate limits. If a background build dies unexpectedly:

What happens: The process exits. Whatever files were created stay in the folder. Agents notice via last-session returning "incomplete" and will ping you.

Draper pings you on Telegram

"Session on <project> ended before completing. Last activity was <X>. Got Y of Z sections done. Want me to continue from where it left off?"

You reply

"Yes continue from <step>" · or · "Pause, I want to review first"

If you say continue, Draper runs openclaw-project continue with a brief pointing at the last completed step. Fresh Claude Code session, fresh context window, picks up from exactly where the previous one stopped.

Never auto-retries. Agents are forbidden from auto-retrying - it creates infinite loops if the underlying cause is a rate limit. Always asks you first.

Break big tasks into smaller ones preemptively

If you know a task is huge (50-section build, full codebase refactor), ask the agent to split it:

"Draper, this is a big one - plan it in 5 phases so we don't hit context limits. Let's do phase 1 first."

Declare a standing rule

In Telegram

"From now on, on all Sooner projects, always use 16px body font."

The agent you said it to captures it in their MEMORY.md immediately. The Sunday dream cycle will auto-promote to the wiki if you repeat it 3+ times across contexts.

In Claude Code session

Same phrasing works:

"From now on, on Sooner, always left-align primary CTAs in heroes."

Claude Code auto-writes this to PREFERENCES.md in the project folder and logs a PREFERENCE captured marker. The owning agent sees it next time they check the project and can promote to the wiki.

What counts as "standing"

  • "From now on..."
  • "Always..."
  • "Never..."
  • "Going forward..."
  • "Make sure to always..."
  • "Let's standardize on..."

Those phrases all trigger the capture flow. One-off instructions ("make this one yellow") don't.

Correct an agent

When an agent says something wrong

"That's wrong. <the right thing>. Update your memory accordingly."

The agent updates their MEMORY.md, acknowledges the correction, and won't make the same mistake next session.

When the wiki has outdated info

"The brand wiki says X but actually Y. Update the brand wiki."

If it's a small correction, the agent edits directly. If it's a significant change to canonical truth, they'll propose the edit and ask you to approve.

When multiple agents contradict each other

Message Lisa (Sooner) or Elon (Axeed):

"Draper and Dinesh are giving different answers about <X>. Reconcile this and give me the right story."

Pod leads have explicit "reconcile, don't relay" rules.

Promote something to the wiki manually

Usually dream cycles handle this. But if you want to shortcut:

"Draper, add a new brand rule to the wiki: <rule>. Create a file in wiki/sooner/brand/ if needed."

The agent will propose the exact file and content before writing. You approve, they write.

Never do this manually yourself. Let the agent propose and write. They'll add the dates, format it correctly, update the CLAUDE.md pointers if needed. Manual wiki edits can break the auto-loading.

  3. Telegram Iris sends personal/cross-pod brief as DM

  • Patterns observed - rules you repeated, emerging standards
  • Auto-promoted - high-confidence rules already written to the wiki (3+ repetitions)
  • Proposals awaiting approval - medium-confidence items numbered for you
  • Stale items - contradictions or outdated claims to fix

Your reply format

"Approve 1, 3, 4. Reject 2 - I didn't mean that as a rule. Revise 5 to say <X> instead."

Lisa/Elon then write the approved changes to the wiki and confirm.

Iris sends a cross-pod monthly review on the 1st at 9am. Cross-pod rules never auto-promote - always propose. Same approve/reject flow.

When things break

Agent stops responding on Telegram/Slack

terminal openclaw gateway restart

If that doesn't work, tail the logs:

terminal tail -f ~/.openclaw/logs/*.log

Disk filling up

Cost Controller pings you on Telegram when below 20GB. If you want to manually check:

terminal df -h $HOME

Common culprits: ~/.openclaw/delegations/ (old delegation temp folders), ~/Library/Caches/, ~/.Trash.

Backup fails

terminal tail -20 ~/.openclaw/handoff-system/logs/backup.log

Usually a file over 100MB snuck in. Add it to ~/.openclaw/.gitignore and retry:

terminal openclaw-backup

Claude Code hooks stopped firing

Check the project has .claude/settings.local.json:

terminal ls ~/Sooner/<project>/.claude/

If missing, re-create via:

terminal openclaw-project adopt --agent draper --pod sooner --project <name> --dir ~/Sooner/<project> --no-import-history

Dream cycle didn't fire on Sunday

Check launchd:

terminal launchctl list | grep ai.openclaw

If missing, reinstall:

terminal ~/.openclaw/dreams/scheduler/install.sh

Anything else weird

Message Iris:

"Iris, something feels off - <describe>. Can you look into it?"

She owns the full infrastructure. Her job is exactly this.

Edge cases worth knowing

You start Claude Code in a folder that has a background build running

The SessionStart hook will detect the conflict and inject a note into your Claude Code session. Claude will say something like:

"There's a background build still running on this project (pid 12345). Should I stop it first with openclaw-project takeover --dir ., or leave it running?"

You say stop or leave, it acts. No file clobbering.

You want to move a project between pods

Don't. Create a new project in the target pod, copy the files manually. Project ownership is encoded in the folder path and the .openclaw-project.json. Cross-pod moves aren't supported because they'd scramble the memory architecture.

You want two agents to collaborate on one project

Currently one project = one owning agent. For collaboration, have the owning agent coordinate with others via shared memory. E.g., Draper owns a build, but pings Dinesh in channel for backend questions.

You want to share a project with Gabriel

Project folders are local to your Mac. To share, push to a separate git repo. The project management system is about local Claude Code handoff, not collaboration across people. (Future: could change this.)

You want to delete a project entirely

terminal rm -rf ~/Sooner/<project> rm -rf ~/.openclaw/shared-memory/sooner/code-projects/<project>

The backup to GitHub will reflect the deletion on next sync. No agent magic required.

You want to rename a project

Also tricky because the project name is in multiple places. Easiest: delete the project, create a new one with the right name, copy files over.

You paused in the middle of a project for weeks

No problem. The folder persists, memory persists, PROJECT_SUMMARY.md is still there. Open Claude Code in the folder, it auto-loads. Or ask the owning agent:

"Draper, bring me back up to speed on <project>. What was the state when I stopped?"

You want to export a PROJECT_SUMMARY.md to share

It's just a markdown file:

terminal cat ~/Sooner/<project>/PROJECT_SUMMARY.md | pbcopy

Paste wherever.

You want to restrict who can talk to agents in a channel

DMs are already Mo-only. In channels, agents are conservative with non-Mo requests (read-only by default). If you want stricter control, remove the agent from the channel. Easiest lever.

A rule you declared last week isn't being followed

Check:

  1. ~/.openclaw/shared-memory/sooner/dreams/<last-sunday>.md - did last Sunday's dream promote it?
  2. If yes, check ~/.openclaw/wiki/sooner/ - is the rule actually in the wiki?
  3. If no, tell the relevant agent: "You're not following my rule about X. Reinforce this and promote it now."

Iris isn't responding to an infrastructure question

Try rephrasing:

"Iris, you own the OpenClaw infrastructure. <your question>."

Sometimes agents need an explicit reminder of their scope to unlock the right tools.
